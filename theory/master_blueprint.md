# Master Blueprint (Interpretive Layer)

This note condenses the source document `Master Blueprint for Research Paper.docx`
into a project-aligned outline.

## Interpretation status

- Layer: **Interpretive Layer (Vision)**.
- Role: conceptual framing for research direction and hypothesis generation.
- Limitation: this is **not** a causal proof and **not** evidence of consciousness.
- Metric boundaries remain defined in `CLAIMS.md`.

## Core vision claims (conceptual)

- Subjective-like behavior is framed as stronger internal temporal closure under
  noisy environments.
- Closed-loop internal dynamics are described as informational "vortices" in
  contrast to purely reactive input-output mappings.
- The proposed transition from reactive to internally self-sustaining dynamics
  is treated as a measurable systems hypothesis, not a metaphysical claim.

## Translation into this repository (metric layer)

- Operational quantity:
  - `I(H_t ; H_{t+1} | E_t)`
- Implementation focus:
  - feedforward vs recurrent vs transformer comparisons,
  - noise-vs-structure checks,
  - training dynamics tracking with CCI trajectories.
- Threshold policy:
  - `1.0 bit` is used only as an operational threshold in this setup.

## Evidence posture

- Code and results in this repository support **diagnostic dependence analysis**.
- Interpretive language must remain downstream of measured outputs and explicit
  assumptions (`theory/assumptions.md`).
- Any broader claims (clinical, legal, consciousness-related) remain future
  hypotheses pending external validation programs.
